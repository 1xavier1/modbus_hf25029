# HF25029-CDP Modbus Slave 使用说明

**生成时间**: 2026-04-16 17:05:00

## 1. 概述

本项目为 TLT153 实现 HF25029-CDP 信号采集处理模块的 Modbus Slave 功能。

### 主要特性

- **Modbus RTU 从站**: RS-485 接口，115200 8N1
- **Modbus TCP 从站**: Ethernet 接口，端口 502
- **支持功能码**: 0x01, 0x02, 0x03, 0x04, 0x05, 0x06, 0x0F, 0x10
- **数据模拟器**: 无需真实硬件即可测试
- **GPIO配置**: 支持通过JSON配置文件自定义引脚
- **两种实现**: Linux 标准版 和 Linux-RT 实时版

## 3. 编译

### 3.1 编译环境

- **PC 开发环境**: Linux (Ubuntu/Debian)
- **目标板**: TLT153 (ARM Cortex-A)
- **交叉编译器**: `arm-linux-gnueabihf-gcc`

### 3.2 编译步骤

```bash
cd /home/tronlong/Workspace/modbus_hf25029

# 1. 编译公共模块 (使用交叉编译器)
cd common
make clean
make CC=arm-linux-gnueabihf-gcc

# 2. 编译 Linux 标准版从站
cd ../slave
make clean
make CC=arm-linux-gnueabihf-gcc

# 3. 编译 Linux-RT 实时版从站
cd ../slave_rt
make clean
make CC=arm-linux-gnueabihf-gcc

# 4. 编译模拟主站 (PC上运行)
cd ../master_sim
make clean
make CC=gcc
```

### 3.3 编译输出

| 模块 | 二进制文件 | 架构 |
|------|------------|------|
| slave | bin/modbus_slave | ARM |
| slave_rt | bin/modbus_slave_rt | ARM |
| master_sim | bin/master_sim | x86_64 (PC) |

## 4. 部署

### 4.1 上传到目标板

```bash

scp common/gpio_config.json root@10.0.0.127:/root/
```

### 4.2 目录结构 (目标板)

```
/root/
├── modbus_slave          # 主程序
```

## 5. 运行

### 5.1 命令行参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| -m, --mode | 运行模式: rtu, tcp, both | both |
| -d, --device | RTU 设备路径 | /dev/ttyAS2 |
| -b, --baud | RTU 波特率 | 115200 |
| -p, --parity | RTU 校验: n, o, e | n |
| -c, --config | GPIO 配置文件路径 | 内置默认 |
| -v, --verbose | 启用调试输出 | 关闭 |
| -h, --help | 显示帮助 | - |

### 5.2 运行示例

```bash
# TCP 模式 (常用)
./modbus_slave -m tcp -v

# 双模式运行
./modbus_slave -m both -v

# 指定GPIO配置文件
./modbus_slave -m tcp -c /root/gpio_config.json -v

# 仅RTU模式
./modbus_slave -m rtu -d /dev/ttyAS2 -b 115200 -v
```

### 5.3 停止程序

```bash
# 查找进程
ps | grep modbus_slave

# 终止进程
kill -9 <PID>
```

## 6. GPIO 配置

### 6.1 配置文件格式

配置文件为 JSON 格式，可自定义 GPIO 引脚映射：

```json
{
    "comment": "HF25029-CDP GPIO Configuration",
    "version": "1.0",

    "di": {
        "channel_count": 22,
        "channels": [
            {"id": 0,  "gpio": 34,  "name": "PB2",  "label": "DI_CH01"},
            {"id": 1,  "gpio": 129, "name": "PE1",  "label": "DI_CH02"},
            ...
        ]
    },

    "do": {
        "channel_count": 12,
        "channels": [
            {"id": 0,  "gpio": 35,  "name": "PB3",  "label": "DO_CH01"},
            ...
        ]
    }
}
```

### 6.2 字段说明

| 字段 | 说明 |
|------|------|
| gpio | Linux GPIO 编号 |
| name | 引脚名称 (如 PB2) |
| label | 通道标签 (如 DI_CH01) |

### 6.3 查看目标板可用GPIO

```bash
# 查看已导出的GPIO
ls -la /sys/class/gpio/

# 查看GPIOchip信息
ls /sys/class/gpio/gpiochip*

# 读取特定GPIO值
cat /sys/class/gpio/gpio<N>/value
```

## 7. Modbus 寄存器映射

### 7.1 输入寄存器 (功能码 0x04)

| 寄存器 | 功能 | 说明 |
|--------|------|------|
| 30001-30005 | 电位器增益 | 5通道, 0-4095 |
| 30006-30027 | DI锁存状态 | 22通道位标志 |
| 30028-30032 | RS485长度 | 5通道, 0-10字节 |
| 30033-30082 | RS485数据 | 5×10字节 |
| 30083-30087 | RS485状态 | 5通道位标志 |
| 30088-30090 | RS232长度 | 3通道, 0-256字节 |
| 30091-30858 | RS232数据 | 3×256字节 |
| 30859-30864 | RS232状态 | 6通道位标志 |

### 7.2 保持寄存器 (功能码 0x03/0x06/0x10)

| 寄存器 | 功能 | 说明 |
|--------|------|------|
| 40001 | 设备地址 | 1-247 |
| 40002 | 波特率 | 0-6 |
| 40020 | 保存参数 | 写入0x55AA保存 |
| 40021-40042 | DI清除 | 清除锁存 |
| 40055-40057 | RS232发送长度 | 3通道 |
| 40058-40825 | RS232发送数据 | 3×256字节 |

### 7.3 线圈 (功能码 0x01/0x05/0x0F)

| 地址 | 功能 | 说明 |
|------|------|------|
| 00001-00012 | DO输出 | 12通道 |

### 7.4 离散输入 (功能码 0x02)

| 地址 | 功能 | 说明 |
|------|------|------|
| 10001-10022 | DI输入 | 22通道 |

## 8. 数据模拟器

程序内置数据模拟器，无需真实硬件即可测试：

| 数据类型 | 模拟方式 |
|----------|----------|
| DI (22路) | 随机 0/1，每 100ms 变化 |
| DO (12路) | 记录主站写入值 |
| 电位器 (5路) | 正弦波 0-4095，周期 10s |
| RS485 (5路) | 模拟帧，每 20ms 更新 |
| RS232 (3路) | 256 字节缓冲 |

## 11. 故障排除

### 11.1 TCP 端口绑定失败

```
[TCP] bind: Permission denied
```

**解决**: 502 是特权端口，需要 root 权限：
```bash
sudo ./modbus_slave -m tcp
```

### 11.2 GPIO 初始化失败

```
[GPIO] Failed to export DI pin 34 (PB2)
```

**说明**: GPIO 已被系统或其他程序占用，程序会使用模拟数据继续运行，不影响 Modbus 通信。

**解决**: 检查设备树配置或使用配置文件指定其他可用 GPIO。

### 11.3 连接失败

```bash
# 检查网络
ping 10.0.0.127

# 检查端口
netstat -an | grep 502
```
