# HF25029-CDP Modbus Slave 使用说明

**生成时间**: 2026-04-16 16:27:00

## 1. 编译

### 1.1 编译环境

- **PC 开发环境**: Linux (Ubuntu/Debian)
- **目标板**: TLT153 (ARM Cortex-A)
- **交叉编译器**: `arm-linux-gnueabihf-gcc`

### 1.2 编译步骤

```bash
cd /home/tronlong/Workspace/modbus_hf25029

# 1. 编译公共模块
cd common
make clean && make

# 2. 编译 Linux 标准版从站
cd ../slave
make clean && make

# 3. 编译 Linux-RT 实时版从站
cd ../slave_rt
make clean && make

# 4. 编译模拟主站 (PC上运行)
cd ../master_sim
make clean && make
```

### 1.3 编译输出

| 模块 | 二进制文件 | 架构 |
|------|------------|------|
| common | libcommon.a | 静态库 |
| slave | bin/modbus_slave | x86_64 (PC) / ARM (交叉编译) |
| slave_rt | bin/modbus_slave_rt | x86_64 (PC) / ARM (交叉编译) |
| master_sim | bin/master_sim | x86_64 (PC) |

### 1.4 交叉编译 (在 PC 上编译目标板程序)

```bash
# 设置交叉编译器
export CROSS=arm-linux-gnueabihf-
export CC=${CROSS}gcc

# 编译 slave
cd slave && make clean && make

# 编译 slave_rt
cd ../slave_rt && make clean && make
```

## 2. 运行

### 2.1 Linux 标准版从站

```bash
# 启用 RTU + TCP 双模式
./slave/bin/modbus_slave -m both

# 仅启用 RTU
./slave/bin/modbus_slave -m rtu -d /dev/ttyAS2 -b 115200

# 仅启用 TCP
./slave/bin/modbus_slave -m tcp -v

# 启用调试模式
./slave/bin/modbus_slave -m both -v
```

### 2.2 Linux-RT 实时版从站

```bash
# 启用 RTU + TCP 双模式
./slave_rt/bin/modbus_slave_rt -m both

# 启用 RTU 并设置参数
./slave_rt/bin/modbus_slave_rt -m rtu -d /dev/ttyAS2 -v
```

### 2.3 模拟主站 (PC上测试用)

```bash
# TCP 主站测试
./master_sim/bin/master_sim -t tcp -h 192.168.1.100 -p 502

# RTU 主站测试
./master_sim/bin/master_sim -t rtu -d /dev/ttyUSB0 -b 115200

# 指定轮询间隔
./master_sim/bin/master_sim -t tcp -i 50  # 50ms 间隔

# 指定测试次数
./master_sim/bin/master_sim -t tcp -c 100  # 运行 100 次后退出

# 详细输出
./master_sim/bin/master_sim -t tcp -v
```

## 3. 命令行参数

### 3.1 从站参数 (slave / slave_rt)

| 参数 | 说明 | 默认值 |
|------|------|--------|
| -m, --mode | 运行模式: rtu, tcp, both | both |
| -d, --device | RTU 设备路径 | /dev/ttyAS2 |
| -b, --baud | RTU 波特率 | 115200 |
| -p, --parity | RTU 校验: n, o, e | n |
| -v, --verbose | 启用调试输出 | 关闭 |
| -h, --help | 显示帮助 | - |

### 3.2 主站参数 (master_sim)

| 参数 | 说明 | 默认值 |
|------|------|--------|
| -t, --type | 类型: tcp, rtu | tcp |
| -h, --host | TCP 主机地址 | 192.168.1.100 |
| -p, --port | TCP 端口 | 502 |
| -d, --device | RTU 设备路径 | /dev/ttyUSB0 |
| -b, --baud | RTU 波特率 | 115200 |
| -a, --addr | 从站地址 | 1 |
| -i, --interval | 轮询间隔(ms) | 100 |
| -c, --count | 测试次数(0=无限) | 0 |
| -v, --verbose | 详细输出 | 关闭 |

## 4. Modbus 寄存器映射

### 4.1 输入寄存器 (功能码 0x04)

| 寄存器 | 功能 | 说明 |
|--------|------|------|
| 30001-30005 | 电位器增益 | 5通道, 0-4095 |
| 30006-30027 | DI锁存状态 | 22通道位标志 |
| 30028-30032 | RS485长度 | 5通道, 0-10字节 |
| 30033-30082 | RS485数据 | 5×10字节 |
| 30083-30087 | RS485状态 | 5通道位标志 |
| 30088-30090 | RS232长度 | 3通道, 0-256字节 |
| 30091-30858 | RS232数据 | 3×256字节 |
| 30859-30864 | RS232状态 | 6通道位标志 |

### 4.2 保持寄存器 (功能码 0x03/0x06/0x10)

| 寄存器 | 功能 | 说明 |
|--------|------|------|
| 40001 | 设备地址 | 1-247 |
| 40002 | 波特率 | 0-6 |
| 40020 | 保存参数 | 写入0x55AA保存 |
| 40021-40042 | DI清除 | 清除锁存 |
| 40055-40057 | RS232发送长度 | 3通道 |
| 40058-40825 | RS232发送数据 | 3×256字节 |

### 4.3 线圈 (功能码 0x01/0x05/0x0F)

| 地址 | 功能 | 说明 |
|------|------|------|
| 00001-00012 | DO输出 | 12通道 |

### 4.4 离散输入 (功能码 0x02)

| 地址 | 功能 | 说明 |
|------|------|------|
| 10001-10022 | DI输入 | 22通道 |

## 5. 数据模拟器

程序内置数据模拟器，无需真实硬件即可测试：

| 数据类型 | 模拟方式 |
|----------|----------|
| DI (22路) | 随机 0/1，每 100ms 变化 |
| DO (12路) | 记录主站写入值 |
| 电位器 (5路) | 正弦波 0-4095，周期 10s |
| RS485 (5路) | 模拟帧，每 20ms 更新 |
| RS232 (3路) | 256 字节缓冲 |

## 6. 硬件连接 (TLT153)

| 接口 | 设备节点 | 说明 |
|------|----------|------|
| RS-485 | /dev/ttyAS2 | 默认 RTU 设备 |
| Ethernet | eth0 | TCP 端口 502 |
| GPIO | sysfs 或 mmap | DI/DO 控制 |

## 7. 故障排除

### 7.1 编译错误

```bash
# 安装依赖
sudo apt install build-essential gcc g++ make

# ARM 交叉编译
sudo apt install gcc-arm-linux-gnueabihf
```

### 7.2 连接失败

```bash
# 检查网络
ping 192.168.1.100

# 检查端口
netstat -an | grep 502
```

### 7.3 权限问题

```bash
# 添加用户到 dialout 组
sudo usermod -a -G dialout $USER
# 重新登录

# 或使用 sudo 运行
sudo ./modbus_slave -m both
```

### 7.4 TCP 端口绑定失败

502 是特权端口，需要 root 权限：

```bash
# 使用 sudo
sudo ./modbus_slave -m tcp

# 或修改为非特权端口 (修改代码中 TCP_PORT 为 1502)
```

## 9. 注意事项

1. **GPIO 初始化失败**: 在 PC 上运行时 GPIO 初始化会失败，这是正常的。程序会自动使用模拟数据继续运行。

2. **实时性能**: 如果需要高精度实时性能，请使用 `slave_rt` 版本 (Linux-RT 内核)。

3. **多从站**: 同一网络上可以运行多个从站实例，每个使用不同的设备地址 (通过 40001 寄存器配置)。

4. **数据一致性**: RTU 和 TCP 共享相同的数据模拟器，任意一个接口写入的数据可以被另一个接口读取。
