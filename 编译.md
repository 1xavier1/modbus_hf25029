# HF25029-CDP Modbus Slave 编译指南

## 1. SDK 目录结构

根据开发手册，LinuxSDK 目录结构如下：

```
LinuxSDK/
├── build.sh                    # 编译脚本
├── device/                     # 设备配置
├── kernel/                     # 内核源码
├── u-boot/                     # U-Boot 源码
├── buildroot/                 # Buildroot 根文件系统
├── out/                       # 编译输出
│   └── t153/
│       └── tlt153-minievm-nand/
│           └── buildroot/
│               └── buildroot/
│                   └── host/bin/   # 交叉编译器
└── 4-软件资料/
    └── Demo/                   # 示例程序 ← modbus_hf25029 应放在这里
        ├── base-demos/         # 基础示例
        ├── linux-rt-demos/    # Linux-RT 示例
        └── ...
```

## 2. 放置位置

### 推荐位置

将 `modbus_hf25029` 目录放置在 SDK 的 Demo 目录下：

```
LinuxSDK/
└── 4-软件资料/
    └── Demo/
        └── modbus_hf25029/    ← 放置在这里
            ├── common/         # 共用模块
            ├── slave/          # Linux 标准版
            ├── slave_rt/       # Linux-RT 实时版
            └── master_sim/    # 模拟主站
```

### 操作步骤

```bash
# 假设 LinuxSDK 在 /home/user/T153/aiot-t153-linux-v1.0/

# 复制项目到 Demo 目录
cp -r modbus_hf25029 /home/user/T153/aiot-t153-linux-v1.0/4-软件资料/Demo/

# 或创建软链接（推荐，方便开发时同步更新）
ln -s /path/to/modbus_hf25029 /home/user/T153/aiot-t153-linux-v1.0/4-软件资料/Demo/modbus_hf25029
```

## 3. 编译方式

### 方式一：独立编译（推荐）

本项目**不需要与 LinuxSDK 一起编译**。应用层程序应独立编译，使用 SDK 提供的交叉编译器：

```bash
# 1. 设置交叉编译器路径
export PATH=/home/user/T153/aiot-t153-linux-v1.0/out/t153/tlt153-minievm-nand/buildroot/buildroot/host/bin:$PATH

# 2. 确认编译器可用
arm-linux-gnueabihf-gcc --version

# 3. 编译 Linux 标准版
cd /home/user/T153/aiot-t153-linux-v1.0/4-软件资料/Demo/modbus_hf25029/slave
make

# 4. 编译 Linux-RT 实时版
cd /home/user/T153/aiot-t153-linux-v1.0/4-软件资料/Demo/modbus_hf25029/slave_rt
make

# 5. 编译模拟主站（在 PC 上运行）
cd /home/user/T153/aiot-t153-linux-v1.0/4-软件资料/Demo/modbus_hf25029/master_sim
make CC=gcc
```

### 方式二：集成到 Buildroot（可选）

如需将程序打包进根文件系统，可作为 Buildroot 外部包：

```
LinuxSDK/
└── buildroot/
    └── external/pkg/
        └── modbus_hf25029/
            ├── modbus_hf25059.mk
            └── modbus_hf25059/
                └── ...源码
```

但这需要修改 Buildroot 配置，不推荐新手使用。

## 4. 编译输出

编译成功后，二进制文件位置：

| 版本 | 文件位置 | 说明 |
|------|----------|------|
| Linux 标准版 | `slave/bin/modbus_slave` | ARM 二进制 |
| Linux-RT 版 | `slave_rt/bin/modbus_slave_rt` | ARM 二进制 |
| 模拟主站 | `master_sim/bin/master_sim` | x86 PC 二进制 |

## 5. 部署到目标板

### 通过 SCP 传输

```bash
# 将二进制文件复制到目标板
scp slave/bin/modbus_slave root@192.168.1.100:/root/

# 或使用 USB 存储复制
```

### 在目标板上运行

```bash
# 添加执行权限
chmod +x modbus_slave

# 运行
./modbus_slave -m both

# 后台运行
nohup ./modbus_slave -m both > /var/log/modbus_slave.log 2>&1 &
```

## 6. Makefile 说明

各模块的 Makefile 支持以下目标：

```makefile
# 标准编译（使用默认 gcc）
make

# 清除编译产物
make clean

# 交叉编译（需设置 CROSS 和 CC）
make CROSS=arm-linux-gnueabihf- CC=arm-linux-gnueabihf-gcc
```

## 7. 编译常见问题

### 问题 1: 交叉编译器路径未设置

```
arm-linux-gnueabihf-gcc: command not found
```

**解决**：确保 SDK 编译输出的 host/bin 目录在 PATH 中：

```bash
export PATH=/path/to/LinuxSDK/out/t153/tlt153-minievm-nand/buildroot/buildroot/host/bin:$PATH
```

### 问题 2: 找不到库文件

```
cannot find -lpthread
```

**解决**：Makefile 中已正确指定 `-lpthread -lrt`，确保使用正确的交叉编译器。

### 问题 3: 权限不足

```
Permission denied
```

**解决**：
```bash
chmod +x bin/modbus_slave
# 或重新编译
make clean && make
```

## 8. 与 SDK 编译系统的关系

| 项目 | 是否需要 SDK 编译 | 说明 |
|------|-------------------|------|
| U-Boot | 需要 | 必须通过 `./build.sh` 编译 |
| Kernel | 需要 | 必须通过 `./build.sh` 编译 |
| Buildroot | 需要 | 必须通过 `./build.sh` 编译 |
| **应用层程序** | **不需要** | 独立编译，使用 SDK 的交叉编译器 |

本项目的编译**独立于 SDK**，只需：
1. SDK 已完成一次完整编译（生成交叉编译器）
2. 设置 PATH 指向交叉编译器
3. 直接 `make` 即可

## 9. 完整编译流程示例

```bash
#!/bin/bash
# compile_modbus.sh

# SDK 路径
SDK_DIR=/home/user/T153/aiot-t153-linux-v1.0

# 设置交叉编译器
export PATH=$SDK_DIR/out/t153/tlt153-minievm-nand/buildroot/buildroot/host/bin:$PATH

# 项目路径
PROJECT_DIR=$SDK_DIR/4-软件资料/Demo/modbus_hf25029

# 编译 Linux 标准版
echo "=== Building Linux Standard Version ==="
cd $PROJECT_DIR/slave
make clean
make

# 编译 Linux-RT 版本
echo "=== Building Linux-RT Version ==="
cd $PROJECT_DIR/slave_rt
make clean
make

# 复制到部署目录
echo "=== Copying to deploy ==="
mkdir -p $SDK_DIR/deploy
cp $PROJECT_DIR/slave/bin/modbus_slave $SDK_DIR/deploy/
cp $PROJECT_DIR/slave_rt/bin/modbus_slave_rt $SDK_DIR/deploy/

echo "=== Done ==="
ls -la $SDK_DIR/deploy/
```
